# Стратегія розгалуження репозиторію

У межах розробки веб-системи для продажу авторських прикрас з конструктором дизайну використовується стратегія розгалуження **GitHub Flow**.

## Загальний опис стратегії

GitHub Flow базується на використанні однієї основної гілки `main`, яка містить актуальний та стабільний стан проєкту. Усі зміни та доповнення до функціональності реалізуються в окремих робочих гілках, що створюються від гілки `main`.

Кожна робоча гілка призначена для виконання окремої задачі або логічного блоку робіт, наприклад реалізації функціоналу, оновлення документації або виправлення помилок.

## Правила роботи з гілками

- гілка `main` використовується як основна та стабільна;
- для кожної нової задачі створюється окрема гілка від `main`;
- після завершення роботи зміни з робочої гілки інтегруються в `main`;
- видалення робочих гілок після злиття є рекомендованим.

## Переваги обраної стратегії

Використання GitHub Flow забезпечує:
- просту та зрозумілу структуру репозиторію;
- поетапне додавання функціональності;
- зручний контроль змін у проєкті;
- зниження ризику конфліктів під час розробки.

Обрана стратегія є доцільною для невеликих веб-проєктів та добре підходить для індивідуальної розробки.
